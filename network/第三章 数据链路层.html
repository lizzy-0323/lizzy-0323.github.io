# 数据链路层[[计算机网络]]

# 数据链路层的功能[[第三章 数据链路层]]

  • 封装成帧
  • 差错控制
  • 流量控制
  • 透明传输

# 封装成帧

目的：解决帧定界、帧同步问题 方式：字符计数法、字符填充、零比特填充（硬件容易实现，适用于同步链路）

# 差错控制

# 奇偶校验码

n-1位信息、1位校验，来保证实现奇校验和偶校验

# 循环冗余码

特点：没有纠错功能 双方约定一个r+1位的多项式，生成r位的冗余码，传输的数据为m+r位

# 海明码

特点：有纠错能力 将位数分为校验位和信息位，可以得知哪一位出错

# 流量控制与可靠传输

  • 停止等待
  • 滑动窗口
  • SR 选择重传(不可以累计确认) 常见的可靠传输组合： 单帧滑动窗口-停止等待 多帧滑动窗口-GBN 多帧滑动窗口-SR

# 介质访问控制

任务：广播信道上的信道划分

# 信道划分介质访问控制(静态）

  • FDM
  • TDM-> STDM:不固定分配时间
  • WDM:光的FDM
  • CDM:码分多路复用，不同站的码片正交，1发送原码，0发送反码，写成-1。

# 随机访问介质控制（动态）

# ALOHA协议

每个站点都不进行检测，随意发送，一旦没收到，一段时间后就再次重发 时隙aloha：每个站只能在一个时隙最开始时发送帧

# CSMA协议（carrier sense mutiple access）

空闲情况下立即发送，忙情况下监听

# CSMA/CD（collision detection）
  • 适用于半双工网络
  • 工作流程：先监听，一旦监听到了空闲就发送，否则就一直监听，一旦检测到了碰撞，就发送一个拥塞信号，让所有用户知道，然后执行退避算法。
  • 争用期：2t
  • 最小帧长：传播时延数据传输速率2
  • 算法流程：确定争用期->确定参数k（不大于10时，为重传次数）->确定基本退避时间->超过16次时报告出错
# CSMA/CA（coliision avoid）

用于无线局域网的协议 所有的站发送完之后，要等待一个帧间间隔inferframe space才能发送下一帧，在这期间必须得到上一个帧的回应，否则认为出现了碰撞 算法流程：检测到信道空闲，等待DIFS之后，再发送整个数据帧，否则执行退避，一旦信道忙，就停止计时，信道空闲，再执行退避，当等待时间变为0，发送整个帧

# 令牌传递协议

轮询的方式，有令牌才能发，适合于负载高的信道

# 局域网（local area network）

常见的局域网结构：星形网、环形、总线型

# 以太网[[局域网]]

局域网最典型的实现，网卡：与外界进行串行，与内部并行，每个网卡有一个MAC地址，长6字节，用12个十六进制数来表示 特点：填充，由于CSMA/CD算法限制，最短帧长是64字节，去掉首尾，需要将内容填充至46字节。 类型：100BASE-T，100mb/s基带信号的星形拓扑以太网，1Gb/s的状态下可以支持半双工和全双工，10Gb/s的状态下只工作在全双工方式。

# IEEE 802.11(无线局域网)

最小构件：BSS，基本服务集 一个基本服务集，通过AP连接到一个分配系统DS

# 广域网

由节点交换机和它们的链路来组成 广域网与局域网的区别：广域网涉及到了三层，包含了网络层，很多节点是点到点的链接

# PPP协议

定义：使用串行线路通信的面向字节的协议，包含NCP（网络控制协议）和LCP 特点：只支持全双工链路、只支持点到点的通信，没有帧编号和确认机制来实现可靠传输

# HDLC协议（high-level data link control）

采用0比特填充法，是一种面向比特流的链路协议

# 数据链路层设备

# 网桥

工作在MAC子层，处理的对象是帧 特点：可以隔离碰撞域，但容易出现广播风暴

# 以太网交换机

是一个多端口的网桥，即插即用，每个端口都与一个主机相连 特点：可以实现虚拟局域网
