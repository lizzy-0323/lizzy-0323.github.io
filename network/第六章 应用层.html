# 第六章 应用层

[[计算机网络]]

# 网络应用模型

# C/S模型

客户是服务请求方，服务器是服务提供方 特点：

  • 各计算机的地位不平等
  • 客户机之间不能相互通信
  • 可扩展性差

# P2P模型

没有固定的服务器和客户端划分 特点：

  • 减轻了服务器计算压力
  • 扩展性好
  • 占内存和网络资源过多

# 应用层的应用

# DNS系统

域名服务器：按照等级排：根域名服务器、顶级域名服务器、权限域名服务器 本地域名服务器：查找时先查找本地域名服务器 域名解析方式：递归查询、迭代查询 主机向本地域名服务器查询时，采用递归查询；本地域名服务器向根域名服务器查询时，采用迭代查询 运输层协议为UDP

# FTP协议

特点：打开两个连接，数据连接为20端口，控制连接为21端口 运输层协议为TCP

# 电子邮件

电子邮件的三个重要组成部分：邮件服务器，用户代理（UA),协议

  • 用户代理：提供接口，来让用户撰写邮件，显示邮件，就是邮件应用程序
  • 邮件服务器：发送和接收邮件
  • 邮件协议：SMTP,POP3 邮件的发送过程：
  • 发信人通过用户代理来撰写邮件，用户代理用SMTP将邮件发给发送端邮件服务器
  • 邮件服务器通过SMTP进程与接收端邮件服务器建立TCP连接并发送邮件
  • 取消连接，当接收端服务器收到邮件后，就存入收信人的信箱
  • 收信人打算收信时，调用用户代理，将邮件从接收端服务器取回 MIME:多用途网际邮件扩充，使得邮件可以传输多个国家的文字 SMTP:推送协议，首先定期的检查邮件，建立连接，然后进行邮件传送 POP3:拉协议，用户读取邮件时，用户代理向邮件服务器发出请求 IMAP:为用户提供了创建文件夹等权利，并且用户可以一次之查看一部分邮件 目前用户浏览器与邮件服务器之间的邮件发送更多的是基于HTTP

# HTTP协议

特点：

  • 面向事务
  • 无连接：每次连接只处理一个请求，结束后连接取消，但后来也实现了长连接，当然采用session和cookie实现了记录 万维网的内核组成：
  • URL，统一资源定位符
  • HTTP,超文本传输协议
  • HTML,文档结构的标记语言 访问资源的流程：
  1. 用户使用URL，基于HTTP协议与web服务器建立连接，然后发送请求报文
  2. web服务器解析URL,找到对应的文件资源，并将资源返回给web浏览器，以HTTP响应报文的形式
  3. 结束连接，释放资源 HTTPS：采用SSL加密HTTP的通信内容，采用对称和非对称两种加密形式
