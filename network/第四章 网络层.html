# 路由算法[[网络层的功能]]

# 距离-向量算法 RIP协议[[路由算法]]

基本思想：所有节点定期讲他们的路由选择表传给与之相连的节点，包括每条路径的目的地，以及路径的距离 更新路由选择表的两个原则：

  • 收到一个新的路由地址
  • 发来的路由信息中，有一个到达某个目的地的路由，有比当前的路由更小的代价
  • 发来的路由信息中，有目的网络，且下一跳的地址相同时，直接更新 缺点：可能遇到路由环路、网络出现故障时收敛很慢，最大距离为15（16为不可达） 特点：应用层协议，上层使用UDP

# OSPF协议[[路由算法]]

基本思想：当链路状态变化时，一个节点向所有节点发送路由信息 发送信息的原则：

  • 只有当链路状态发生变化时，路由器才向所有路由器发送信息
  • 链路状态的度量包括：度量、时延、带宽、费用等 计算方法：Diijkstra算法 特点 ：网络层协议

# 层次路由[[路由算法]]

当网络范围扩大时，分为内部和外部两个网关协议： 内部协议：RIP,OSPF 外部协议(EGP)：BGP[[BGP]] 将一个区域划分为多个as（自治系统），AS指单一技术管理下的一组路由器

# IPV4 [[网络层协议]]

# 常见的字段的意义和功能

  • 首部长度：占4位，表示可以使用的最大十进制数是15，但是单位是4B,所以最大为60B
  • 总长度：首部和数据之和的长度，单位是字节，一共16位，最大长度为65535B
  • 标识：仅用于重组数据报
  • 标志：MF=1，表示后面还有分片，MF=0,表示后面没有分片，DF=0表示允许分片
  • 生存时间：TTL,8位，路由器在转发前，先将TTL减1，一旦为0，就丢弃
  • 协议：占8位，表示运输层协议
  • 首部校验和：16位，只校验首部
  • 源和目的地址：4B

# 数据报分片

分片的原因：数据链路层能承载的最大数据量（MTU)为1500B，片偏移的倍数是8的倍数

# 地址划分

A类、B类、C类

# 网络地址转换（NAT)[[地址划分]]

将专用网络地址转换为公用地址，私有地址只用于LAN。 原理为ip：端口号映射 NAT工作在传输层、ARP工作在网络层

# 子网划分与CIDR[[地址划分]]

# 子网划分

将原本的主机号划分为子网号和主机号，主机号全0和全1都无法使用，全0为子网的网络号，全1为广播地址 子网掩码：用于告诉一个网络子网的位数，并且得到这个网络的网络号 路由寻址过程：

  1. 路由器收到一个ip地址，首先查看是否直接交付
  2. 如果不能，则对路由表各个表项按位与，看是否有能交付的
  3. 交给默认路由
# CIDR(classless interdomain routing)

取消abc划分的方式，但仍旧使用掩码的思想，可以用cidr的思想将路由表内的表项进行路由聚合

# ICMP报文

网络层的应用 ICMP报文的种类：差错报告报文、询问报文 对一个差错报文不再发送差错报文，对分片的ip数据报，只对第一个发送差错报文，对特殊地址不发送ICMP差错报告报文 ICMP的应用：ping（工作在应用层），traceroute（工作在网络层）

# IPV6协议

特点：

  1. 以16个字节来表示地址，用冒号分隔16位
  2. 不允许分片
  3. 支持单播、多播、任播

# IP组播

采用IGMP协议管理组播组，组播时只需要发送一份数据，也需要组播路由器。 IP组播数据报的特点：

  • 前4位为1110，D类地址。
  • 组播地址只能用于目标地址，不提供可靠交付 **IGMP的内容工作阶段：
  • 每次有新主机加入时，向组播组的组播地址发送一个IGMP报文，组播路由器收到之后，将组成员关系转发给其他组播路由器
  • 组播路由器周期性查看成员是否响应，一旦有响应就认为还在活跃，无响应就不再将该组成员关系转发给其他路由器

# 移动IP(隧道技术)

实现机理：

  • 移动节点在本地时，采用传统的TCP/IP通信
  • 移动节点在外地时，使用固定IP通信，移动节点向本地代理注册一个位置地址，本地代理打开一条隧道，将IP分组转给转发地址处
  • 在转发地址处，接触隧道封装，送到移动节点
  • 每次更换地址，只需要本地代理更新注册的转交地址

# 网络层设备

路由器：有多个输入输出接口的计算机，实现了网络模型的下三层 分组转发部分由三部分组成：交换结构、输入端口、输出端口
