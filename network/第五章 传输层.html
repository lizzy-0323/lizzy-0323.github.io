# 第五章 传输层

[[计算机网络]]

# 传输层的功能

  • 提供进程间逻辑通信
  • 复用和分用
  • 差错检验
  • 提供两种传输协议

# UDP协议

# UDP的优点

  • 无需建立连接，减少时延
  • 分组首部开销少，TCP有20B的首部开销，UDP只有8B
  • 不提供拥塞控制，不限制主机的发送速率
  • 提供一对多，多对多交互通信
  • 面向报文 UDP的应用：DNS,SNMP

# UDP的首部格式

  • 源端口 16位
  • 目的端口
  • UDP长度
  • 检验和 总共8B

# UDP校验

计算校验和时，在数据报前加入一个12B的伪首部，伪首部仅用来计算校验和，UDP计算校验和时，检查首部和数据部分。

# TCP协议

# TCP的特点

  • 面向连接
  • 一对一
  • 提供可靠传输
  • 提供全双工通信 UDP报文的长度由发送应用进程决定，TCP报文的长度由接收方的窗口值和当前网络的拥塞程度来决定

# TCP首部

组成：有20B的固定部分，以及4N的可变长度部分 源端口和目的端口：各2B 序号：4B 确认号：4B 数据偏移：数据段距离报文初始段还有多远 保留： URG:紧急指针 ACK:确认号 PSH:推送位，用于尽快交付 RST:出现严重差错时，用于释放连接 SYN：用于请求连接或者连接接受 FIN：用于释放连接 窗口：占2B 校验和：同样需要加上伪首部 紧急指针：在URG=1时有意义，用于指出紧急数据有多少字节 MSS:指定数据字段的最大长度

# TCP三次握手与四次握手

三次握手

  1. 首先先发送SYN=1给server，client进入同步发送状态
  2. server发回确认，为该连接分配缓存和变量，服务器进入同步收到状态
  3. client此时可以发送数据，进入established状态 四次握手
  4. 客户机想关闭连接时，发送FIN=1的报文段
  5. 服务器收到后，发出确认，此时客户机到服务器的连接就释放了
  6. 服务器再发送FIN=1的报文段，此时服务器进入LAST-ACK状态
  7. 客户端收到之后，发出确认，将ACK置为一

# TCP可靠传输

实现方式：

  • 序号
  • 确认
  • 超时重传
  • 冗余ACK：当接收方收到失序的报文时，就发送一个冗余ACK，表明下一个期待的报文序号，如果收到三个冗余ACK，则可以认为已丢失，进行快速重传

# TCP流量控制

接收方每次收到报文，就回复目前的rwnd值，让发送方调整

# TCP拥塞控制

拥塞控制与流量控制的区别：拥塞控制并不只针对端点，而针对整个网络。 拥塞算法：

  • 慢开始：每收到一个报文，cwnd就加倍，直到阈值
  • 拥塞避免：达到阈值后，每收到一个报文，cwnd就加1，一旦遇到拥塞，将阈值设置为当前的cwnd的一半，然后将cwnd设置为1
  • 快重传和快恢复：快重传就是指连续收到三个冗余ACK直接重传，快恢复就是指遇到三个ACK后，将阈值设置为cwnd的一半，并把cwnd设置为阈值改变后的数值，然后执行加法增大而不是慢开始

发送方的发送窗口大小如何确认？： min（cwnd，rwnd）
