# 进程管理

# 进程的概念和特征

进程的概念：进程=程序段+数据段+PCB，PCB是进程存在的唯一标志，资源分配的基本单位 进程的特征：

  • 动态性
  • 并发性
  • 独立性
  • 异步性：结果不可预见 进程的状态：运行态、就绪态、等待态

# 进程控制

将进程的pcb放入指定的队列

# 进程通信

  • 共享存储
  • 消息传递
  • 管道通信：只能采用半双工操作

# 线程

线程：CPU调度的基本单位，分为用户级线程和内核级线程，模型有一对多、一对一、多对多

# cpu调度

  • 作业调度：从外存调入作业
  • 中级调度：将不能运行的进程调到外存，将具备条件的进程调到就绪队列，类似于一种交换策略
  • 进程调度：是操作系统中最基本的一次调度，从就绪队列里选进程执行 调度的基本准则：
  • 周转时间：从作业提交到作业完成的时间
  • 等待时间：处于等待状态的时间
  • 带权周转时间：周转时间/作业实际运行时间
  • 响应时间：从用户提交请求到系统首次响应的时间 调度算法：
  • FCFS：先来先服务
  • SJF:运行时间最短的，运行时间最短的优先
  • 优先级调度算法：
  • 高响应比优先：响应比=等待时间+要求服务时间/要求服务时间
  • RR算法：抢占式
  • 多级反馈队列调度：新任务先放入第一个队列中，当时间片用完后，放入第二个队列

# 死锁

死锁产生的四个条件：

  • 不剥夺条件：非抢占式调度
  • 请求并保持：每个进程都有请求，但都不放弃自己已有的资源
  • 互斥条件：资源为临界资源
  • 循环等待：等待形成了一个循环
