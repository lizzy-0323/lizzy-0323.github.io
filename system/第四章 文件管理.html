# 文件管理 [[操作系统]]

# 文件系统基础

文件的概念：以计算机硬盘为载体的存储在计算机上的信息集合 文件的分类：流式文件：无结构，记录式文件：有结构 文件打开的过程： ![[Pasted image 20220723102120.png]] 无结构文件：源程序文件、代码文件等 记录式文件：

  • 顺序文件：记录在磁带上
  • 索引文件：利用索引表进行查找 目录结构：目录表包含文件名和索引节点的对应结构，找到inode节点号之后，就可以去找到inode节点. 硬链接和软链接： 硬链接：inode节点接受fd指针的指向，当count=0时，系统会删除节点 **软链接：共享链接存储的时一个路径名，当真实的inode节点消失时，软链接会出现问题

# 连续分配

每个文件在磁盘上连续，拥有最少的寻道时间和寻道数，容易实现外部碎片

# 链接分配

隐式链接：每个文件有一个磁盘块链表每个盘块都有指向下一个盘块的指针 显式链接：每个磁盘一张FAT，标记了各个盘块的连接关系，目录里仅有一个起始块号，FAT在系统启动时就会被读入内存

# 索引分配

每个文件都有一个索引块，支持了直接访问

# 文件实现

文件存储器空间划分：每个逻辑卷可以是一个物理盘，也可以进一步划分，每一个逻辑卷由目录区和文件区组成 文件存储器空间管理（空闲块的管理）：空闲表、空闲链表、位示图

# 磁盘组织与管理

磁盘调度算法：

  • FCFS
  • SSTF：寻找与当前磁头所在磁道最近的磁道
  • SCAN：不仅要考虑距离，还要顺应磁头的移动方向
  • C-SCAN：循环扫描 影响最长的时间：寻道时间，磁头移动到指定磁道要花费的时间。 ，你，模拟， 磁盘的管理过程：
  1. 磁盘初始化：首先将磁盘划分扇区，称为低级格式化，其次是创建文件系统，进行逻辑格式化
  2. 引导块：运行时，首先运行自举程序，初始化CPU、寄存器、设备控制器和内存，然后找到磁盘上的操作系统内核启动，装入内存
  3. 坏扇区的处理方式：采用小型计算机系统接口，维护一个坏块链表，操作系统是不能修复坏块的。
